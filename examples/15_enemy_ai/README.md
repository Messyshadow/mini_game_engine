## 核心概念

### 什么是有限状态机（FSM）？

有限状态机是游戏AI最经典的实现方式。核心思想很简单：

- 敌人在任一时刻只处于**一种状态**
- 每种状态有自己的**行为逻辑**
- 满足特定条件时**切换状态**

比现实生活中的红绿灯：红灯→绿灯→黄灯→红灯，每个灯有固定的规则。

### 状态转换图

```
                 ┌─────────────────────────┐
                 │                         │
    ┌──────┐  发现玩家  ┌──────┐  进入攻击范围  ┌──────┐
    │ 巡逻 │ ────────→ │ 追击 │ ────────────→ │ 攻击 │
    │Patrol│ ←──────── │Chase │ ←──────────── │Attack│
    └──────┘  丢失目标  └──────┘  目标离开范围  └──────┘
        │                  │                      │
        ↕                  │                      │
    ┌──────┐           ┌──────┐                   │
    │ 空闲 │           │ 受击 │ ←─────────────────┘
    │ Idle │           │ Hurt │      被击中
    └──────┘           └──────┘
        │                  │
        └────────→ ┌──────┐ ←─────────────────────┘
           HP<=0   │ 死亡 │      HP<=0
                   │ Dead │
                   └──────┘
```

### 各状态行为

| 状态 | 行为 | 转换条件 |
|------|------|----------|
| Idle | 原地不动，等待计时器结束 | 计时结束→Patrol；发现玩家→Chase |
| Patrol | 在巡逻范围内来回走动 | 到达端点→Idle；发现玩家→Chase |
| Chase | 向玩家方向跑动 | 进入攻击范围→Attack；玩家跑远→Patrol |
| Attack | 停下来执行攻击 | 攻击结束→Chase/Patrol |
| Hurt | 硬直不动 | 硬直结束→Chase |
| Dead | 播放死亡动画，不再行动 | （终态） |

### 近战型 vs 远程型

| 特性 | 近战型 (robot4) | 远程型 (robot8) |
|------|-----------------|-----------------|
| 追击行为 | 直接冲向玩家 | 保持偏好距离（~200px） |
| 攻击范围 | 65px | 250px |
| 太近处理 | 继续贴脸 | 后退拉开距离 |
| 攻击动画 | punch | shoot |
| 移动速度 | 较快 (160) | 较慢 (120) |

## 关键代码解析

### AIConfig 配置结构

```cpp
struct AIConfig {
    EnemyType type = EnemyType::Melee;
    
    // 感知距离
    float detectionRange = 250.0f;   // 多远能发现玩家
    float loseRange = 350.0f;        // 多远会丢失玩家（> detection，防抖动）
    float attackRange = 70.0f;       // 多近会发起攻击
    
    // 巡逻范围
    float patrolMinX, patrolMaxX;    // 来回走的左右端点
    float idleDuration = 1.5f;       // 走到端点停多久
    
    // 速度
    float patrolSpeed = 80.0f;       // 巡逻走路速度
    float chaseSpeed = 180.0f;       // 追击跑步速度
};
```

### 为什么 loseRange > detectionRange？

这是一个经典的**滞后设计**（Hysteresis），防止敌人在检测边界上反复切换状态。

如果两个距离一样，玩家站在刚好250px处，敌人会：发现→追→玩家稍退→丢失→回头→又发现→又追… 来回抖动。

设成 detection=250, lose=350 后，敌人发现玩家要在250以内，但丢失要到350以外，中间有100px的缓冲区。

### 远程敌人的三段距离逻辑

```cpp
if (dist < retreatDistance) {
    // 太近了（<100px），往后退
    moveDirection = 远离玩家方向;
} else if (dist > preferredDistance) {
    // 太远了（>200px），靠近
    moveDirection = 接近玩家方向;
} else {
    // 合适距离（100~200px），停下来射击
    moveDirection = 0;
    if (冷却好了) 发起攻击;
}
```
